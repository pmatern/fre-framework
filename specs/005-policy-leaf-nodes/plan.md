# Implementation Plan: Extended Policy Rule Leaf Nodes

**Branch**: `005-policy-leaf-nodes` | **Date**: 2026-03-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-policy-leaf-nodes/spec.md`

## Summary

Extend `PolicyRule::Variant` with 13 new leaf node types enabling richer event matching:
tag substring/membership/existence/numeric comparisons, first-class event field predicates
(`event_type`, `tenant_id`, timestamp age), evaluator score range, and pipeline health
inspection (`StageIsDegraded`, `EvaluatorWasSkipped`, `EvaluatorReasonIs`). Changes are
confined to `rule_engine.hpp` and `rule_engine.cpp`. Numeric tag parsing uses
`std::from_chars` (no-throw, no new dependencies). TDD: unit + integration tests written
before implementation.

### Failure Mode Analysis

| What can fail | Blast radius | Degradation strategy |
|---|---|---|
| `RuleEngine::evaluate` encounters unknown variant index | Per-event, one pipeline call | Impossible by construction — `std::visit` is exhaustive; compile error if branch missing |
| `std::from_chars` parse failure on non-numeric tag | Per-rule match only | Returns `false` immediately; no exception; pipeline continues normally |
| Tag absent from event | Per-rule match only | Returns `false`; no error or degradation |
| Evaluator/stage not present in `stage_outputs` | Per-rule match only | Returns `false`; existing stage outputs unaffected |
| Future-timestamped event in `EventOlderThan`/`EventNewerThan` | Per-rule match only | Negative age detected; returns `false`; no exception |

**Shuffle sharding**: N/A. Rule evaluation is stateless, per-event, and executes on the
tenant's already-assigned strand. No new resource allocation crosses tenant boundaries.

**Noisy-tenant isolation**: N/A. CPU cost per rule is bounded by rule count × O(n) tag
scan. Each tenant runs on dedicated strands; no cross-tenant contention introduced.

**P99 latency budget**: 13 new `if constexpr` branches add at most a few hundred
nanoseconds to policy stage evaluation. Well within the 20ms `PolicyStageConfig::timeout`
and the 300ms pipeline latency budget.
